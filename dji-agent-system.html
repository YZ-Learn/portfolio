2. Agent 核心 Agent/react_agent.py

整个系统的中枢。create_agent 把模型、提示词、工具、中间件拼成一个 LangGraph 可执行图。 

from langchain.agents import create_agent
from model.factory import chat_model
from Agent.tools.middleware import monitor_tool, log_before_model, report_prompt_switch

class ReactAgent():
    def __init__(self):
        self.agent = create_agent(
            model=chat_model,
            system_prompt=load_system_prompts(),
            tools=[rag_summarize, get_weather, get_user_id,
                   get_current_month, get_user_location,
                   fetch_external_data, fill_context_for_report],
            middleware=[monitor_tool, log_before_model, report_prompt_switch]
        )

执行时走 stream 流式模式，逐 chunk 吐内容，前端逐字渲染。 用户看到的是打字机效果，不是整段等。 

def execute_stream(self, query: str):
    input_dict = {'messages': [('user', query)]}
    for chunk in self.agent.stream(input_dict, stream_mode='values', context={'report': False}):
        last_message = chunk['messages'][-1]
        if last_message.content:
            yield last_message.content.strip() + '\n'

🔌 fill_context_for_report — 中间件触发器

最巧妙的设计。这个工具本身不返回值，但在中间件里注入 context['report'] = True 标记。 report_prompt_switch 中间件检测到这个标记后，自动把 Agent 的系统提示词 从"客服提示词"切换成"报告生成提示词"。

MiddlwareDynamic Prompt

中间件是关键设计。LangChain 支持 @wrap_tool_call、 @before_model、@dynamic_prompt 三种钩子。 我三个都用上了： 

# 工具调用时：记录日志 + 注入 report 标记
@wrap_tool_call
def monitor_tool(request, handler):
    logger.info(f"[monitor_tool]执行工具：{request.tool_call['name']}")
    result = handler(request)
    if request.tool_call['name'] == "fill_context_for_report":
        request.runtime.context['report'] = True  # ← 注入标记
    return result

# 模型推理前：检查标记，动态切换提示词
@dynamic_prompt
def report_prompt_switch(request):
    if request.runtime.context.get("report", False):
        return load_report_prompts()   # ← 切到报告模式
    return load_system_prompts()        # ← 默认客服模式

4. RAG 流水线 RAG/

专业文档 → 分片 → 嵌入 → 存 Chroma → 检索 → LLM 重总结。 支持自动加载 PDF/TXT，MD5 去重。

📄 文档加载器 (vector_store.py)

遍历 大疆相关/ 目录，自动识别 .txt 和 .pdf。 用 RecursiveCharacterTextSplitter 分片（200字符/片，20字符重叠）。 每次加文档前算 MD5，重复内容自动跳过。

🔎 检索 + 重总结 (rag_service.py)

查 Chroma 取 top-3 相关文档，拼成 context， 用定制 PromptTemplate 让 LLM 基于参考资料生成答案。 再加了一层 StrOutputParser 确保返回纯文本。 

# RAG 链：Prompt → LLM → 纯文本
chain = prompt_template | print_prompt | model | StrOutputParser()

def rag_summarize(self, query: str) -> str:
    context_docs = self.retriever_docs(query)
    context = ""
    for i, doc in enumerate(context_docs, 1):
        context += f"[参考资料{i}：{doc.page_content}]"
    return self.chain.invoke({"user_input": query, "reference": context})

5. 核心流程：从用户提问到回答

👤 用户提问 → Streamlit 接收 → ReactAgent 解析 → 🧠 LLM 思考
← 需要资料？→ → 🔧 调工具 → 📊 拿结果 → 🔄 再思考
✅ 信息够了 → 💬 LLM 生成回答 → ⚡ 流式输出 → 📱 用户看到

最复杂的是报告生成路径：获取 ID → 获取月份 → 调 fill_context_for_report → 中间件注入标记 → 系统自动切提示词 → 拉飞行记录 → 查保养资料 → LLM 写完整报告。 整个过程Agent 自主决策调用顺序，不是写死的。 

# 报告生成的工具调用链（Agent 自主决定，不是硬编码）
1. get_user_id()             → "1005"
2. get_current_month()      → "2025-06"
3. fill_context_for_report()  → # 中间件注入 report=True
4. fetch_external_data(..., ...) → # 拉飞行记录
5. rag_summarize("无人机 保养") → # 查保养资料
                               → "天文学家张大冰之大疆使用情况报告..."
